一、简介

1 什么是数据库

最流行的关系型数据库

数据库系统（DB,Database System）包含

  • 数据库（DB）
  • 数据库管理系统(DBSM)
  • 应用开发工具
  • 管理员及用户

SQL语言：结构化查询语言（Structurede Query Language）包含

  • DDL 数据定义语言
  • DML数据操作语言（增删改）
  • DQL数据查询语言（查）
  • DCL数据控制语言

2 mysql相关操作

  • 登录退出
  • 查看版本

SELECT VERSION();

  • 打开mysql并打开D数据库

mysql -uroot -P -D db_name

  • mysql默认结束符号是；或者\g,不输入是不会结束的
  • 可以通过help或者\h或者?加上相关关键字来查看手册
  • y语句不想执行，后加\c代表取消语句
  • 常用sql语句
  • 注意上面这些关键字其实都有问题，在LINX在关键字要大写，库名 表名 字段要小写
  • 数据库，表名，字段名等尽量不要使用关键字，如果一定要，要加上反引号`
  • #或者 –是注释

DDL（data defination language）数据库定义语言，主要是建表、删除表、修改表字段等操作

3 数据库相关操作

  • 创建数据库

CREATE DATABASE|SCHEMA [IF NOT EXISTS]database_name;--中括号代表可选

可以指定编码方式 CREATE DATABASE|SCHEMA [IF NOT EXISTS]database_name DEFAULT CHARACTER SET [=]'charset';

  • 查看全部数据库

SHOW DATABASES;

  • 查看指定数据库信息

SHOW CREATE DATABASE db_name;

  • 修改指定数据库编码方式

ALTER DATABASE db_name ;

  • 查看上一步操作的警告信息

SHOW WARNINGS

  • 打开某个数据库

USE db_name

  • 显示当前数据库

SELECT DATABASE()|SCHEMA();

  • 删除指定数据库

DROP DATABASE [IF NOT EXISTS] db_name;

4 数据表

数据表是数据库最重要组成，由行(row)列(column)组成

  • 列至少有一列，行可以为0
  • 创建数据表语法规范
  • 查看已有表

SHOW TABLES [FROM|IN] database_name [LIKE 'pattern' WHERE expr];

  • 查看指定表详细信息

SHOW CREATE TABLE tbl_ name;

  • 查看表结构

DESC tbl_name; DESCRIBE tbl_name; SHOW COLUMNS FROM tbl_name;

  • 查新表中所有记录（注意和上一条区别）

SELECT * FROM tbl_name

  • 删除表

DROP TABLE IF EXISTS tbl_name;

  • 向表中插入一条记录

INSERT [INTO] tbl_name(id,username,……) VALUES(1,'KING',……)

  • 添加删除字段
  • 添加删除默认操作
  • 修改字段
    • 改类型属性位置用MODIFY
    • 改名字类型属性位置用CHANGE
  • 添加删除主键

（如果主键有自增，先去掉自增才能删除主键）

  • 添加删除索引
    • 主键索引
    • 唯一索引 ：避免重复的列
    • 常规索引：key或者index设置
    • 全文索引：特定引擎下有，快速定位数据
  • 修改表名

5 数据类型

  • 数值型

    • 整数

      TINYINT（4）括号里数字代表显示长度，对于TINYINT来说-128-127是上下限所以最大就支持4位

    • 浮点 、定点

  • 字符串型

    • CHAR和VARCHAR不同，默认会用空格填充，检索时CHAR中字符尾端的空格会消失
    • CHAR定长，相当于空间换时间，查询更快
    • TEXT不能有默认值，且不区分大小写
    • ENUM可以使用设定的值插入，或者1、2、3代替，另外枚举类会自动过滤空格
    • SET插入后会自动排序

  • 时间日期型

    • time

    • ​ time/data

      YY-MM-DD YYMMDD 70-99转换成1970~1999 00~69 2000~2069

    • datatime

    • TIMESTAMP

    插入CURRENT_ TIMESTAMP/插入NULL/什么也不写 都可以得到当前系统日期和时间

    • YEAR

​ 1901 2055 范围

6 约束条件

  • unsigned 无符号（符号占一位）

  • zerofill 零填充

  • not null 非空

  • default默认值

  • primary key 标志着记录的唯一性 一个表只能有一个主键，==自动禁止为空==，值不能重复

    • 最好找一个无意义的数据类型
    • 如果不设置默认值，插入数据必须赋值
    • 可以设置复合主键，只要有一个数据不同就不算重复

  • unique 唯一性 值不能重复，但是null除外

  • AUTO_INCREMENT 自动增长 只能用于数值列 配合索引使用

    • 一个表只能有一个自动增长，且后面要加关键字key(索引)

  • FOREGINE KEY 外键约束 ——保证数据的一致性和完整性

    • 只有INNODB引擎支持外键

    • 创建外键

      [CONSTRANT 外键名称]FOREIGN KEY(字段名称) REFERENCES 主表 （字段名称）

      • 子表的外键字段和主表的主键字段类型要相似;如果是数值型要求-致，并且无符号也要求一致;如果求类型一致，长度可以不同
      • 如果外键字段没有创建索引，MySQL会自动帮我们添加索引
      • 子表的外键关联的必须是父表的主键

  • 动态添加/删除

    ALTER TABLE tbl_ name ADD CONSTRAINT[外键名称] FOREIGN KEY(外键字段) REFERENCES 主表(主键字段);

    ALTER TABLE tbl_ name DROP FOREIGN KEY fk name;

    动态添加之前要确保没有脏值

  • 外键约束的其他操作

    • CASCADE 级联操作，父表删除或者更新，子表也删除或者更新
    • SET NULL 父表删除更新时设置子表外键列为NULL
    • NO ACTION | RESTRICT 拒绝对父表删除或者更新操作
  • COMMENT 注释

7 MYSQL存储引擎

  • MyISAM

    • 产生frm表结构文件 MYD存储数据文件 MYI索引文件

    • 可以在创建表的时候指定数据文件和索引文件的存储位置,只有MyISAM表支持，DATA DIRECORY [=]数据保存的绝对路径，INDEX DIRECTORY [=]索引文件保存的绝对路径

    • MyISAM单表最大支持的数据量2的64次方条记录
      每个表最多可以建立64个索引

    • 如果是复合索引,每个复合索引最多包含16个列,索引值最大长度是1000B

    • MyISAM引擎的==存储格式==

      • 定长( FIXED静态) 是指字段中不包含VARCHAR/TEXT/BLOB

      • 动态( DYNAMIC ) 只要字段中包含了VARCHAR/TEXT/BLOB

        ENGINE=My ISAM ROW_FORMAT=FIXED#将动态转换为定长

      • 压缩( COMPRESSED )

  • INNODB

    • 设计遵循ACID模型,支持事务,具有从服务奔溃中恢复的能力,能够最大限度保护用户的数据

    • 原子性( Atomiocity)
      一致性 ( Consistency)
      隔离性( Isolation )
      持久性( Durability )

    • 支持行级锁,可以提升多用户并发时的读写性能

    • 支持外键,保证数据的一致性和完整性

    • InnoDB拥有自己独立的缓冲池,常用的数据和索引都在缓存中
      对于INSERT. UPDATE DELETE操作 , InnoDB会使用一种change buffering的机制来自动优化,还可以提供一致性的读,并且还能够缓存变更的数据,减少磁盘I/O ,提高性能

    • 创建InnoDB表之后会产生两个文件

      • .frm表结构文件

      • ibd,数据和索引存储表空间中

      • 所有的表都需要创建主键,最好是配合，上AUTO INCREMENT,也可以放到经常查询的列作为主键

主要是数据库增删改三种操作

  • 不指定字段(需要按照建表时候的字段顺序给每个字段都赋值)

  • 指定字段（不赋值的要有自增或默认值）

INSERT tbl_name SET 字段名称 = 值

  • INSERT SELECT 从表中引入字段

四、事务

原则：ACID

A(Atocimity)原子性：要么都成功要么都失败

A有800块，B有200块，A给B转账200块,转账后要么成功要么失败，两人同时发生财产变化

C(Consistency)一致性：事务前后数据完整性必须保持一致，开始总量和结束总量一致

开始总共是1000.转账完还是

I隔离性(Isolation)：事务的隔离性是多个用户并发访问数据库时,数据库为每一一个用户开启的事务 ，不能被其他事务的操作数据所干务之间要相互隔离。

如果C也给B转账，和A给B转账先后顺序要分清楚，此时会出现

  • 脏读 C转账给B时读取了A还未转账给B时B的余额，导致转账后B的余额错误
  • 不可重复读：读一次就行了，B每时每刻都在收钱，数据在不断变化
  • 虚读（幻读）：读到了新插入的行（正在转钱的D）

D持久性(Durability)：事务一旦提交则不可逆，被持久化到数据库中!

钱要么没转出去，要么转出去了，不存在中间状态

4 sql注入

  • sql注入利用了sql语言关键字漏洞，比如使用or或–

  • 解决

    根本解决：字符串拼接问题，使用preparedStatement

    PreparedStatement是Statement的子接口,它的实例对象可以通过调用
    Connection.preparedStatement(sq|)方法获得,相对于Statement对象而
    言:

    • PreperedStatement可以避免SQL注 入的问题。
      一 Statement会使数据库频繁编译SQL ,可能造成数据库缓冲区溢出。
    • PreparedStatement可对SQL进行预编译,从而提高数据库的执行效率。
      -并且PreperedStatement对于sq|中的参数 ，允许使用占位符的形式进行
      替换,简化sq|语句的编写。
