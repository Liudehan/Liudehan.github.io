static静态、参数传递、单例模式、工厂模式

静态

  • 静态最基本的作用就是定义类的时候某个变量使用了静态，被引用的时候可以对相同的属性统一定义，直接类名.属性名就可以用，叫做静态变量或者类变量，同样的，方法也可以是静态的。静态不涉及对象，所以内部不能用this/super/yes
  • 静态不能访问常态，但是常态可以访问静态，main函数是静态的，他之所以能访问非静态类是因为new了新对象，涉及内存知识，暂不深究。
  • 对于变量（写在类里面，不在方法里）使用static就可以直接被其他类调用，不用实例化，将其称为类变量，不使用static就要set get

    顺便说下代码块

  • 代码块也分静态和动态，静态代码块被main函数中多个对象调用只会执行一次
  • 假设有一个Person类，定义main函数,调用Person类的时候，可以做一个匿名的内部子类写在 new Person()后面，这个类里面可以继承Person的属性，可以重写Person的方法，注意这个子类虽然没有构造器但是要用代码块代替构造器的功能。

单例模式

实例化只做一次，原因就是new一个很复杂的对象需要大量时间，当需要批量new同一个类的对象的时候，可以采用单例模式，节省内存，也不容易出错。简单说无论你什么时候调用，都是同一个对象。

  • 饿汉模式
    做一个私有化的构造器，在类内部做一个私有的静态的new对象，然后使用getInstance做一个共有静态的方法，返回私有的对象，以后调用的时候直接用类名.getInstance.
  • 懒汉模式
    做一个私有化的构造器，在类内部做一个私有的静态的null的变量，然后使用getInstance做一个共有静态的方法，判断是null就返回私有的对象，以后调用的时候直接用类名.getInstance，在第一次以后，都会有相同的对象. 参考资料：https://blog.csdn.net/qq_41457412/article/details/80598787

    工厂模式

    FactoryMethod模式是设计模式中应用最为广泛的模式，在面向对象的编程中，对象的创建工作非常简单，对象的创建时机却很重要。FactoryMethod解决的就是这个问题，它通过面向对象的手法，将所要创建的具体对象的创建工作延迟到了子类，从而提供了一种扩展的策略，较好的解决了这种紧耦合的关系。简单来说，当两个开发人员合作，一个写接口一个写实现时，如何才能让写实现的开发人员不受写接口人员频繁的改动影响。

JAVA封装中构造方法与setget

构造方法

  • 内容来源于百度百科
    构造方法是一种特殊的成员方法,它的特殊性反映在如下几个方面:
  1. 构造方法作用：（1）.构造出来一个类的实例 （2）.对构造出来个一个类的实例（对象）初始化。
  2. 构造方法的名字必须与定义他的类名完全相同，没有返回类型，甚至连void也没有。
  3. 主要完成对象的初始化工作，构造方法的调用是在创建一个对象时使用new操作进行的。
  4. 类中必定有构造方法，若不写，系统自动添加无参构造方法。接口不允许被实例化，所以接口中没有构造方法。
  5. 不能被static、final、synchronized、abstract和native修饰。
  6. 构造方法在初始化对象时自动执行,一般不能显式地直接调用.当同一个类存在多个构造方法时，java编译系统会自动按照初始化时最后面括号的参数个数以及参数类型来自动一一对应。完成构造函数的调用。
  7. 构造方法分为两种：无参构造方法 有参构造方法
    构造方法可以被重载。没有参数的构造方法称为默认构造方法，与一般的方法一样，构造方法可以进行任何活动，但是经常将他设计为进行各种初始化活动，比如初始化对象的属性。
  8. 构造代码块：
  • 作用：给对象进行初始化，对象一建立就执行，而且优先于构造函数执行
  • 构造代码块和构造函数的区别：
    　　构造代码块是给所有不同对象的共性进行统一初始化，构造函数是给对应的对象进行初始化
  1. 子类继承父类中，
  • 子类的实例化过程
  • 构造方法不能被子类继承
  • 子类创建对象时，会先去创建父类的对象。
  • 默认是去调用父类的无参构造方法。
  • 子类构造方法中，第一行默认是super()
  • 为什么子类中第一行会默认有super()
  • 因为他继承父类的成员使用，使用前这些成员必须初始化，
  • 而他们是父类的成员，所以，必须通过父类进行初始化。
  • 所以，会先创建一个父类的对象。
  • 当父类没有无参构造方法时
    必须使用this或者super调用其他的构造方法。

10.自定义类中，如果不写构造方法，java系统会默认添加一个无参的构造方法。如果写了一个有参的构造方法，就一定要写无参构造方法。
如果想使用无参的构造方法，就必须手动给出无参构造方法。
建议：一般情况下，我们自定义的类都要手动给出无参构造方法。

搭建本地hexo博客及markdown操作命令

花费好几个小时，终于搭建hexo博客成功，感谢codesheep的b站教程,liudehan的技术博客，作为一个小白，也是跳了很多坑，在这里记录一下搭建流程和自己进的坑。

一 准备工具

  • Git：版本控制器（或者CMD）

  • Node：提供服务器端JavaScript开发环境

  • GitHub：需注册一个账号

以上这些直接搜索下载即可，该注册的注册，github绑定的账号密码邮箱都记好，后面搭建过程要用。

二 hexo安装

1、安装nodejs和安装git，网站搜索下载，无坑可跳。

2、git bash here或者cmd输入指令，安装hexo。

执行以下代码：
npm install -g hexo-cli
注意第一个坑，如果你是用的gitbash,那么你在哪里新建的git就会决定你之后的博客存放位置，记好这个位置，免得之后忘记了。同理,cmd一般都有默认路径，所以如果直接打开cmd执行上面代码，应该会将你的博客放在C：\user下。
安装完成后，执行下面代码检验是否安装成功：
hexo -v

3、进入hexo

确保你的当前指令是在对应文件夹才能成功进入，如果不对你可以通过以下代码实现上下级文件夹的进入与退出。
//进入 $ cd yourfile
//退出 $ cd ..\

4、安装hexo服务器

执行命令:$ npm install hexo-server --save
这样安装好hexo服务器以后，我们就可以启动服务器了。执行命令$ hexo server，然后在浏览器输入网址http://127.0.0.1:4000/ ,就可以正常访问了
